Ubuntu 환경에서 **Kind (Kubernetes in Docker)**를 설치하는 과정을 단계별로 정리해 드립니다.
Kind는 Docker 컨테이너를 사용하여 쿠버네티스 노드를 실행하기 때문에, Docker 설치가 선행되어야 합니다. 또한 클러스터를 제어하기 위한 kubectl도 필요합니다.
1단계: 필수 패키지 업데이트 및 Docker 설치
Kind는 컨테이너 런타임(Docker) 위에서 작동합니다. 먼저 Docker를 설치합니다.
 * 패키지 목록 업데이트 및 필수 도구 설치
   sudo apt update
sudo apt install -y curl apt-transport-https ca-certificates software-properties-common

 * Docker 설치 (Ubuntu 저장소 이용)
   가장 간편한 방법인 Ubuntu 기본 저장소를 통해 설치합니다.
   sudo apt install -y docker.io

 * Docker 서비스 시작 및 권한 설정
   Docker를 실행하고, 현재 사용자에게 sudo 없이 Docker를 쓸 수 있는 권한을 부여합니다.
   # Docker 자동 실행 설정 및 시작
sudo systemctl enable --now docker

# 현재 사용자를 docker 그룹에 추가 (sudo 없이 docker 명령 사용 위함)
sudo usermod -aG docker $USER

   > 중요: 그룹 권한 변경 사항을 적용하려면 로그아웃 후 다시 로그인하거나, 다음 명령어를 입력하여 현재 세션을 갱신해야 합니다.
   > 
   > newgrp docker
   > 
   > 
2단계: kubectl 설치 (Kubernetes CLI)
Kind로 클러스터를 만들어도 이를 제어하려면 kubectl이 필요합니다.
 * kubectl 바이너리 다운로드 (최신 안정 버전)
   curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"

 * 설치 및 실행 권한 부여
   sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl

 * 설치 확인
   kubectl version --client

3단계: Kind 설치
이제 Kind 도구 자체를 설치합니다.
 * Kind 바이너리 다운로드
   (아래 명령어는 최신 버전 중 하나인 v0.20.0을 예시로 합니다. 필요시 버전을 변경할 수 있습니다.)
   # AMD64 아키텍처 (일반적인 Intel/AMD CPU)
[ $(uname -m) = x86_64 ] && curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-linux-amd64

# ARM64 아키텍처 (라즈베리파이, 맥 M1/M2 VM 등)
[ $(uname -m) = aarch64 ] && curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.20.0/kind-linux-arm64

 * 실행 권한 부여 및 이동
   chmod +x ./kind
sudo mv ./kind /usr/local/bin/kind

 * 설치 확인
   kind --version

4단계: Kind 클러스터 생성 및 확인
설치가 완료되었으니 실제로 쿠버네티스 클러스터를 생성해 봅니다.
 * 클러스터 생성
   기본 이름(kind)으로 클러스터를 생성합니다. (처음 실행 시 노드 이미지를 다운로드하느라 시간이 조금 걸릴 수 있습니다.)
   kind create cluster

   특정 이름으로 생성하고 싶다면: kind create cluster --name my-cluster
 * 클러스터 상태 확인
   클러스터가 정상적으로 생성되었는지 kubectl로 확인합니다.
   kubectl cluster-info --context kind-kind

 * 노드 확인
   kubectl get nodes

   결과가 Ready 상태로 나온다면 설치 성공입니다.
요약 (설치 후 자주 쓰는 명령어)
| 동작 | 명령어 |
|---|---|
| 클러스터 생성 | kind create cluster |
| 클러스터 목록 확인 | kind get clusters |
| 클러스터 삭제 | kind delete cluster |
| 특정 클러스터 삭제 | kind delete cluster --name [이름] |
